# Integration Components - Person C\n\nThis directory contains the frontend components for Person C's portion of DeceptGrid.\n\n## 📁 Files Overview\n\n### Core Components\n- **`Steg.jsx`** - Engineer steganography panel for hiding/extracting messages in images\n- **`Injection.jsx`** - Attacker page 4 for false data injection attacks\n- **`StolenLogin.jsx`** - Attacker page 5 for stolen credential login attempts\n\n### Auto-Refresh Integration\n- **`HoneypotPanel.jsx`** - Sample implementation showing auto-refresh integration\n- **`../hooks/useAutoRefresh.js`** - Custom hooks for real-time log updates\n\n## 🔄 Auto-Refresh Integration for Person B\n\n### Quick Integration\n\nTo add live log updates to your dashboard components, use the provided hooks:\n\n```javascript\nimport { useHoneypotLogs, useAttackLogs } from '../../hooks/useAutoRefresh';\n\nfunction YourDashboardComponent() {\n  // Auto-refresh every 2.5 seconds\n  const { logs, loading, error } = useHoneypotLogs(2500);\n  \n  // Use the logs in your component\n  return (\n    <div>\n      {logs.map((log, index) => (\n        <div key={index}>\n          {log.time} - {log.type} from {log.ip}\n        </div>\n      ))}\n    </div>\n  );\n}\n```\n\n### Available Hooks\n\n1. **`useHoneypotLogs(interval)`** - Person A's honeypot activity\n2. **`useAttackLogs(interval)`** - Person C's attack simulation logs  \n3. **`useMeterStatus(interval)`** - Person B's meter status updates\n\n### Features\n- ✅ Automatic cleanup on component unmount\n- ✅ Error handling and loading states\n- ✅ Configurable refresh intervals\n- ✅ Memory leak prevention\n\n## 🎨 Styling Patterns\n\n### Engineer Components (Green Theme)\n```javascript\n// Card styling\nclassName=\"bg-green-50 rounded-lg p-6 border border-green-200\"\n\n// Button styling  \nclassName=\"bg-green-600 hover:bg-green-700 text-white\"\n\n// Status badge\nclassName=\"bg-green-100 text-green-800 px-3 py-1 rounded-full\"\n```\n\n### Attacker Components (Red/Dark Theme)\n```javascript\n// Background\nclassName=\"min-h-screen bg-gray-900 text-green-400 font-mono\"\n\n// Terminal header\nclassName=\"bg-black border border-red-600 rounded-t-lg\"\n\n// Button styling\nclassName=\"bg-red-700 hover:bg-red-600 text-white\"\n```\n\n## 🔗 API Integration\n\nAll API calls use the centralized constants from `../constants/api.js`:\n\n```javascript\nimport { API, apiCall, apiUpload } from '../../constants/api';\n\n// Standard API call\nconst data = await apiCall(API.ATTACK_LOGS);\n\n// File upload\nconst response = await apiUpload(API.STEG_ENCODE, formData);\n```\n\n## 📱 Component Usage\n\n### In Router Setup\n```javascript\nimport Steg from './pages/integration/Steg';\nimport Injection from './pages/integration/Injection';\nimport StolenLogin from './pages/integration/StolenLogin';\n\n// Use in your router configuration\n<Route path=\"/engineer/steg\" component={Steg} />\n<Route path=\"/attacker/inject\" component={Injection} />\n<Route path=\"/attacker/stolen-login\" component={StolenLogin} />\n```\n\n### Integration with Person B's Dashboard\nThe `Steg` component is designed to be used as a tab/panel within Person B's engineer dashboard. It can be imported and used directly:\n\n```javascript\nimport Steg from '../integration/Steg';\n\nfunction EngineerDashboard() {\n  const [activeTab, setActiveTab] = useState('overview');\n  \n  return (\n    <div>\n      {/* Your tab navigation */}\n      {activeTab === 'steganography' && <Steg />}\n    </div>\n  );\n}\n```\n\n## 🧪 Testing\n\nTo test the components:\n\n1. **Start the backend**: `cd backend && uvicorn main:app --reload`\n2. **Start the frontend**: `cd frontend && npm run dev`\n3. **Test endpoints**: Use the component UIs or tools like Postman\n\n## 📞 Coordination Notes\n\n- **Person A**: Components expect honeypot logs at `/api/honeypot/logs`\n- **Person B**: Sample dashboard panel provided as reference\n- **Backend**: All endpoints documented in `../constants/api.js`\n\nThe auto-refresh functionality enables live demo capabilities where attacks in one tab immediately appear in monitoring dashboards in another tab.
